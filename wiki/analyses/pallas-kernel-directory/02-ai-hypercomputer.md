Catalog of Pallas kernels embedded in Google AI-Hypercomputer's three training/serving stacks: [MaxText](https://github.com/AI-Hypercomputer/maxtext) (Gemma/Llama/DeepSeek/Qwen/Mistral/Kimi reference trainer), [MaxDiffusion](https://github.com/AI-Hypercomputer/maxdiffusion) (Flux/Wan/SDXL/LTX diffusion reference), and [JetStream](https://github.com/AI-Hypercomputer/JetStream) (inference engine, **being archived 2026-02-01** with functionality migrating to [AI-Hypercomputer/tpu-inference](https://github.com/AI-Hypercomputer/tpu-inference)). All are Apache-2.0. Every kernel is `mosaic_tpu` backend — none of these repos ship Mosaic-GPU or Triton kernels. Part of [2026-04-23 Pallas kernel directory](../2026-04-23-pallas-kernel-directory.md).

Attention-kernel selection is standardized across the three repos via an `attention:` config knob routing to one of the kernels below, e.g. MaxText `base.yml`: `attention: 'autoselected' # Supported: autoselected, dot_product, flash, cudnn_flash_te` plus `attention=paged` / `attention=vllm_rpa` for serving.

## 2.1 AI-Hypercomputer/maxtext

Kernels at [`src/maxtext/kernels/`](https://github.com/AI-Hypercomputer/maxtext/tree/main/src/maxtext/kernels) and [`src/maxtext/inference/`](https://github.com/AI-Hypercomputer/maxtext/tree/main/src/maxtext/inference). The perf guide ([`docs/guides/optimization/pallas_kernels_performance.md`](https://github.com/AI-Hypercomputer/maxtext/blob/main/docs/guides/optimization/pallas_kernels_performance.md)) names three families as the memory-bandwidth-bound or structurally-irregular ops MaxText ships custom Pallas for: training attention (Flash/Splash), serving attention (paged & ragged), MoE grouped matmul (Megablox GMM).

## Cross-repo observations

**Vendored / upstream-of lineage:**

- **Splash attention**: `src/maxtext/kernels/attention/splash_attention_kernel.py` and `src/maxdiffusion/kernels/splash_attention/splash_attention_kernel.py` are both vendored from [jax.experimental.pallas.ops.tpu.splash_attention](https://github.com/jax-ml/jax/tree/main/jax/experimental/pallas/ops/tpu/splash_attention). MaxText additionally imports tokamax's splash copy via `config.use_splash_scheduler`. MaxDiffusion switches via `attention_kernel: flash | tokamax_flash | tokamax_ring`. Canonical lineage: **jax-ml/jax upstream → (tokamax / DeepMind) → vendored copies**.

- **Ring attention**: Only MaxDiffusion ships it at [`ring_attention_kernel.py`](https://github.com/AI-Hypercomputer/maxdiffusion/blob/main/src/maxdiffusion/kernels/splash_attention/ring_attention_kernel.py), composing with splash fwd/bwd primitives. Announced 2026-04-16. **This is the integration target for Tokamax ring-attention on TPU within this group.**

- **Paged / ragged paged**: MaxText's [`paged_attention_kernel_v2.py`](https://github.com/AI-Hypercomputer/maxtext/blob/main/src/maxtext/inference/paged_attention_kernel_v2.py) is an explicit vendored copy ("This is a copied version of …ragged_paged_attention.py"). JetStream's [`paged_attention.py`](https://github.com/AI-Hypercomputer/JetStream/blob/main/experimental/jax/inference/kernel/attention/tpu/paged_attention.py) vendors from jax-ml/jax but targets the older non-ragged kernel; its `chunked_prefill_attention` is JetStream's "ragged/unified" equivalent built before MaxText's v2 was vendored.

- **Megablox GMM**: Exists only in MaxText ([`megablox/backend.py`](https://github.com/AI-Hypercomputer/maxtext/blob/main/src/maxtext/kernels/megablox/backend.py)). JAX Pallas implementation of Megablocks ([arXiv:2211.15841](https://arxiv.org/abs/2211.15841)). Already wires a fall-through to `tokamax.ragged_dot(implementation="mosaic")`. No MaxDiffusion/JetStream analogue.

- **SparseCore gather-reduce**: Only in MaxText; uses lower-level `jax.experimental.mosaic` dialects rather than `pallas_call`.

- **Collective matmul**: Only in JetStream's experimental stack; absent from MaxText (which relies on XLA's own AG/RS scheduling) and MaxDiffusion.

**Backend matrix:** every Pallas kernel targets `mosaic_tpu`. **No Mosaic-GPU (SM90/SM100) or Triton kernels** shipped in these three repos — GPU paths fall through to XLA (`jax_flash_attention.py` in MaxText) or to `cudnn_flash_te`. Explicit signal: `validate_gpu_flash_attention` in [layers/attentions.py](https://github.com/AI-Hypercomputer/maxtext/blob/main/src/maxtext/layers/attentions.py) raises on sinks/QK-clip; JetStream collective_matmul file notes collective-permute "doesn't work for GPU well".

**Stability inference:**

- MaxText `src/maxtext/kernels/*` + `src/maxtext/inference/*` — **stable** (no `experimental/` in paths; called out in the perf doc as defaults for Gemma/Llama/DeepSeek/Qwen/Mistral/Kimi training and for `paged` inference configs).
- MaxDiffusion `src/maxdiffusion/kernels/splash_attention/*` — **stable** for `flash`/`tokamax_flash`; ring announced 2026-04-16.
- All JetStream `experimental/jax/inference/kernel/*` — **experimental** (path + README prototype disclaimer). **Repo archives 2026-02-01**; functionality migrated to [tpu-inference](https://github.com/AI-Hypercomputer/tpu-inference).

**Production-config wiring:**

- MaxText `config.attention == "autoselected"` picks splash for Gemma/Llama/DeepSeek/Qwen/Mistral/Kimi training ([configs/models/](https://github.com/AI-Hypercomputer/maxtext/tree/main/src/maxtext/configs/models)).
- MoE configs (DeepSeek v2/v3/v3.2, Llama4, Mixtral, Qwen-MoE, Kimi-K2-1T) route MoE matmuls to Megablox GMM via [layers/moe.py](https://github.com/AI-Hypercomputer/maxtext/blob/main/src/maxtext/layers/moe.py), with `use_tokamax_gmm` toggling the tokamax backend.
- Serving configs under [configs/inference/](https://github.com/AI-Hypercomputer/maxtext/tree/main/src/maxtext/configs/inference) set `attention: paged`, selecting `paged_attention_kernel_v2`.
- MaxDiffusion: `attention_kernel` ∈ `{flash, tokamax_flash, tokamax_ring}` across Flux/Wan2.1/2.2/SDXL/LTX configs.
- JetStream: llama2-7b-only; historical.
