Catalog of Pallas kernels published by Apple AxLearn and several public Google DeepMind repositories. These labs ship the most diverse set of published **non-upstream** Pallas kernels: SSM / linear-attention families (AxLearn is the only public source for Mamba/Mamba2/linear-attention Pallas kernels), a fused GLU kernel (AlphaFold 3), a canonical linear-recurrence LRU scan (RecurrentGemma), and non-LLM sparse attention (GraphCast). [Simply](https://github.com/google-deepmind/simply) and [GraphCast](https://github.com/google-deepmind/graphcast) are included for completeness; both wrap upstream JAX Pallas kernels rather than defining their own. Part of [2026-04-23 Pallas kernel directory](../2026-04-23-pallas-kernel-directory.md).

## 4.1 apple/axlearn

Apple's public training framework. **Single largest source of non-upstream Pallas kernels surveyed.** Ships side-by-side TPU (Mosaic) and GPU (Triton) implementations for flash attention, paged attention, decoding, grouped matmul, plus **exclusive** Pallas kernels for Mamba, Mamba2 (SSD), and linear/RAttention — none of which exist in `jax.experimental.pallas.ops` or tokamax. Splash/Flash dispatch logic documented in [arXiv:2507.05411](https://arxiv.org/pdf/2507.05411).

## 4.2 google-deepmind/recurrentgemma

Publishes a single but clean Pallas LRU scan — the canonical public Mosaic TPU kernel for `h_t = a_t · h_{t-1} + x_t`, backs the Griffin RG-LRU layer. Supports real + complex accumulators, multi-shard correction for sequence-parallel training.

| Kernel | Source path | Backend | Stability | Perf claims | Use case | Callers | Notes |
|---|---|---|---|---|---|---|---|
| `linear_rnn_pallas_kernel` / `lru_pallas_scan` / `pallas_lru` | [recurrentgemma/jax/pallas.py](https://github.com/google-deepmind/recurrentgemma/blob/main/recurrentgemma/jax/pallas.py) | `mosaic_tpu` | stable | (no explicit perf number) | LRU scan backing the Griffin / RecurrentGemma RG-LRU block; fwd + bwd, optional `a_prod` accumulation, sharded-sequence correction | `recurrentgemma/jax/scan.py` (`lru_pallas_scan = pallas.lru_pallas_scan`) → `recurrentgemma/jax/layers.py` RecurrentBlock → `recurrentgemma/jax/griffin.py` | Supports `complex_lib.Complex` accumulators (rare — most Pallas kernels are real-only). Configurable via `pallas_max_seq_block_size` (default 256), `pallas_min_seq_block_size` (16), `pallas_pad_seq_to_min_block_size`, `pallas_pad_last_dim_to_128`. `multi_shard_correction` composes the kernel with per-shard prefix products under `sequence_shard_index`. **Direct ancestor of AxLearn's `mamba_kernels.py`.** UNIQUE — only public LRU Pallas scan designed from first principles for Griffin-family models |

## 4.3 google-deepmind/simply

`ragged_paged_attention.py` is the **serving wrapper, not a kernel**: imports `from jax.experimental.pallas.ops.tpu import ragged_paged_attention`. Contribution: autotuner heuristic + paging scheduler, not a new Pallas kernel.

## 4.5 google-deepmind/alphafold3 (pinned to tag v3.0.1)

Two GPU kernel families — a Triton/jax_triton flash-attention (not Pallas, but adjacent) and a **Pallas-on-Triton fused Gated Linear Unit** kernel. **The GLU kernel is the production reference for "what a fused matmul+activation+matmul Pallas kernel looks like"** — tokamax's GLU currently falls back to XLA on TPU, making this the most informative public benchmark.

### Uniqueness vs upstream / tokamax

| Kernel family | Uniqueness status |
|---|---|
| AxLearn Mamba1 (`mamba_kernels.py`) | **UNIQUE.** No public Mamba Pallas kernel in `jax.experimental.pallas.ops`, tokamax, TorchTitan, MaxText, or Penzai. Apple derives from RecurrentGemma's LRU scan but specializes for Mamba's `(state_dim, inner_dim, seq_len)` layout |
| AxLearn Mamba2 / SSD (`ssd_kernels.py`) | **UNIQUE.** Only public Mamba2 Pallas implementation. Two-level chunk/subchunk recomputation is distinct from upstream JAX reference |

### Patterns worth elevating to concept pages

1. **Block-sparse offset masks for paged attention** — AxLearn `tpu_paged_attention_kernel.py` precomputes `(n_kv_blocks, n_kv_blocks)` offsets into unmasked blocks, turning sliding-window into dense-load-over-sparse-offset. Generalizable.
2. **Online softmax with logit sink** — AxLearn splash extension; init `m` with sink, add `exp(sink - m_final)` to denominator at normalize. Applies to any flash-attention variant.
3. **Dropout-inside-kernel from prng_key + block indices** — AxLearn splash; pair with known Pallas bug (`key<pl>` lowering) requiring prefetch.
4. **Two-level chunk/subchunk recomputation** — AxLearn SSD kernel; store coarse-grained carries, recompute fine-grained states. Applicable to any linear-recurrence Pallas kernel.
5. **Grouped program-IDs to minimize L2 footprint** — AlphaFold3 `_get_best_pids` / `_get_group_cache_usage`; reusable in any GPU Pallas GEMM.
6. **DMA-overhead-bytes heuristic for block-size autotuning** — Simply's `autotune_block_sizes` treats DMA setup as ~0.5 MiB "virtual bytes" and balances against padding overhead. Cheap, portable for any paged TPU kernel.
7. **Multi-shard sequence-parallel correction for linear recurrences** — RecurrentGemma `multi_shard_correction`; run local scan per shard, broadcast final `h` and `a_prod`, recompose. Applies to any sequence-parallel SSM.
