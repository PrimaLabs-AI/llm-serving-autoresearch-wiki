Catalog of every Pallas kernel in the two ecosystem anchors: [jax-ml/jax](https://github.com/jax-ml/jax) (first-party reference kernels under `jax.experimental.pallas.ops.*`) and [openxla/tokamax](https://github.com/openxla/tokamax) (DeepMind kernel library). Everything downstream either vendors, extends, or dispatches to the kernels tabulated here. Part of [2026-04-23 Pallas kernel directory](../2026-04-23-pallas-kernel-directory.md).

## 1.1 jax-ml/jax

User-facing Pallas kernels live under `jax/experimental/pallas/ops/` split by backend: `tpu/` (Mosaic TPU), `gpu/` (mix of Pallas-Triton and Pallas-Mosaic-GPU — distinguished by whether the file imports `jax.experimental.pallas.triton` or `mosaic_gpu`). Everything is **experimental** per JAX's stability policy — no file carries a stable API guarantee and several self-describe as pedagogical.

## 1.2 openxla/tokamax

Per-op directory structure under `tokamax/_src/ops/<op>/` with backend-agnostic `base.py` (XLA reference), user-facing `api.py`, and Pallas backends named `pallas_mosaic_tpu*.py`, `pallas_mosaic_gpu*.py`, `pallas_triton*.py`. README warns "heavily under development"; supported-kernel matrix is: GPU `dot_product_attention`, `gated_linear_unit`, `layer_norm`; GPU+TPU `ragged_dot`; TPU-only `linear_softmax_cross_entropy_loss`.

## Uncategorized / noted-but-not-a-kernel

- JAX: `jax/_src/pallas/mosaic_gpu/*` ([core.py, lowering.py, primitives.py, pipeline.py, helpers.py, torch.py](https://github.com/jax-ml/jax/tree/main/jax/_src/pallas/mosaic_gpu)) — compiler/runtime layer for Mosaic-GPU, NOT kernels. These define how `plgpu.*` primitives lower to PTX/MLIR.
- Tokamax `base.py` per op — backend-agnostic Op definitions + XLA reference implementations; not Pallas kernels.
- Tokamax `*_common.py` files — shared `Config` dataclasses across SM90/SM100.
- Tokamax `attention/jax_nn.py` — wraps `jax.nn.dot_product_attention` (cudnn path), not Pallas.
- Tokamax `microbenchmarks.pdf` — binary PDF; could not be read inline.
